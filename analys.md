## Анализ выполненной задачи

1. Архитектура приложения разделена на backend и frontend часть, что обеспечивает масштабируемость и удобство разработки. Backend реализован на Go с использованием слоёв (repository -> service -> handler), frontend на React + TS

2. Реализована полноценная система аутентификации и авторизации с использованием JWT-токенов, что позволяет разграничивать доступ пользователей и администраторов

3. База данных PostgreSQL спроектирована с учётом нормализации: таблицы книг, авторов, категорий, пользователей, покупок, аренд и уведомлений связаны через внешние ключи

4. Реализован пользовательский интерфейс с возможностью:
   - просмотра каталога книг
   - фильтрации по категории, автору и году
   - сортировки
   - покупки и аренды книг

5. Реализован административный интерфейс, позволяющий:
   - добавлять, редактировать и удалять книги
   - управлять статусами и доступностью
   - добавлять авторов и категории

6. Добавлена бизнес-логика аренды:
   - разные тарифы (2 недели, 1 месяц, 3 месяца)
   - автоматическое вычисление дат
   - изменение статуса книги

7. Реализован механизм напоминаний, который:
   - периодически проверяет сроки аренды
   - создаёт уведомления о скором окончании аренды

8. Реализована защита данных:
   - хэширование паролей с помощью bcrypt
   - middleware проверки ролей

9. Обеспечена интеграция frontend и backend по REST API
